При работе в режиме неявного (или подразумевающегося) начала транзакции SQL Server автоматически начинает новую транзакцию, как только завершена предыдущая. Пользователь ничего не должен делать, чтобы указать начало транзакции. Транзакция продолжается до тех пор, пока пользователь явно не укажет команду отката (ROLLBACK TRANSACTION) или конца (COMMIT TRANSACTION) транзакции, после чего сервер автоматически начинает новую транзакцию. В итоге генерируется непрерывная цепь транзакций.

После того как для соединения установлен режим подразумевающегося начала транзакции, сервер автоматически заканчивает текущую транзакцию и начинает новую, если встречается одна из следующих команд:

ALTER TABLE – изменение структуры таблицы; CREATE; DELETE; DROP; FETCH; GRANT; INSERT; OPEN; REVOKE; SELECT – выборка данных из таблиц; TRUNCATE TABLE - усечение таблицы; UPDATE.

Для установки режима автоматического определения транзакций используется следующая команда Transact-SQL:

SET IMPLIСIT _TRANSACTION ON
