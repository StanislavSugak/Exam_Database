Репликация – это копирование данных, расположенных на одном сервере, на один или несколько других серверов. SQL Server поддерживает репликацию данных не только на другие SQL Server, но и на серверы предыдущих версий, а также на СУБД сторонних производителей.

Однако все имеющиеся возможности репликации могут быть реализованы только в том случае, если все участники репликации являются серверами SQL Server.

Модель репликации SQL Server базируется на метафоре «опубликуйте и подпишитесь». Терминология репликации для серверов участников процессов тиражирования данных используется термин издатель, дистрибьютер и подписчик. Предполагается, что издатель, подписчик и дистрибьютер запускается на разных SQL Server.

При этом можно сконфигурировать репликацию таким образом, чтобы данные копировались между базами данных одного сервера. В этом случае издатель, подписчик и дистрибьютер будут запускаться на одном компьютере.

Помимо перечисленных терминов, для описания процесса репликации используется понятие публикация и статья. Статья представляется собой таблицу или её часть, выбранную для тиражирования. Когда выбирается часть таблицы, используется либо вертикальное разделение, либо горизонтальное.

Вертикально разделение ограничивается количество колонок в таблице, включенных в репликацию. Горизонтальное разделение устанавливает условие на выбираемые для тиражирования строки. Публикация представляется собой набор статей, копируемых как одно целое между сервером-издателем и сервером-подписчиком. Подписчик может подписаться только на всю репликацию. Подписка на отдельную статью в репликации не поддерживается.
