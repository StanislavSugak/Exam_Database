Модель безопасности SQL Server – состоит из трех разных категорий, которые взаимодействуют друг с другом:

1. Принципалы – субъекты, которые в SQL Server имеют разрешение на доступ к определенной сущности.

2. Защищаемые объекты. Ресурсы, доступ к которым регулируется системой авторизации базы данных.

3. Разрешения. Каждый защищаемый объект имеет связанные с ним разрешения, которые могут быть предоставлены принципалу.

Компонент Database Engine может работать в одном из следующих режимов аутентификации:

1) режим аутентификации Windows (Windows Authentication mode). В этом режиме SQL Server полностью доверяет операционной системе при аутентификации пользователей;

2) в смешанном режиме – режим аутентификации SQL Server и Windows – SQL Server and Windows Authentication mode. В этом режиме системы аутентификации Windows и SQL Server существуют независимо друг от друга.

Разрешения для объектов.

Работа с данными и выполнение хранимых процедур требуют наличия класса доступа, называемого разрешениями для объектов.

Для различных объектов применяются разные наборы разрешений:

– SELECT, INSERT, UPDATE и DELETE – эти разрешения могут быть применены для таблицы или представления;

– SELECT и UPDATE – эти разрешения могут быть применены к конкретной колонке таблицы или представления;

– INSERT и DELETE – эти разрешения применяются для таблицы или представления;

– EXECUTE – это разрешение применяется только к хранимым процедурам.

Разрешения для команд T-SQL.

Этот класс разрешений контролирует возможность создания объектов в базе данных, создания самой базы данных и выполнения процедуры резервного копирования. Например, если пользователю необходимо создать представление, администратор базы данных должен предоставить пользователю право на выполнение команды CREATE VIEW (TABLE, RULE, DATABASE) или BACKUP DATABASE(LOG)/
